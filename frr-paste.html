FRR paste → render (xray-core gallery)

Paste a router's show ip route and show ip ospf neighbor output and press Render. The local topology (this router + its neighbors, plus peer–peer links inferred from equal-cost routes) is reconstructed and drawn through the tidy xrayCore API.
🔒 Runs entirely in your browser — your config never leaves the page (no upload, no server).
Supported here: FRR show ip route + show ip ospf neighbor, OSPF, small topologies. BGP / large meshes / other vendors aren't auto-parsed yet — feed the engine state directly instead (see DATA-CONTRACT.md), or fork frr-parse.js as an adapter template.

self node = r1; neighbors become r2, r3, … 
// frr-parse.js turns the pasted text into the engine's config/state…
var r = FRR.build(routeText, neighborText, 'r1');

// …and xrayCore draws it — two clean calls, no globals:
var view = xrayCore.renderTopology('#topo', r.config, { topology: r.topo, trace: r.trace });
view.applyState(r.state);

// zoom into this router's internals (the cylinder):
view.openDeepDive();
Field shapes: see DATA-CONTRACT.md. Topology is inferred from one router's view; for large meshes that one router can't see fully, combine several routers' output. Descriptive only — not a simulator.
The live ping/packet animation is hidden in this demo (traffic-specific); the engine supports it via Seam B trace + ping_ok — see DATA-CONTRACT.md §Seam B.
