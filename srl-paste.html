paste one node's collector state and render its control-plane panel

Run clab-srl-collect.js against a live Nokia SR Linux node (it reads real sr_cli "info from state … | as json") and paste the JSON here. The same single-node panel the FRR collector drives renders it — routing table, BGP table and Best-Path decision — because the engine is vendor-neutral; only the state source differs.

Capture your own from a running lab:
node clab-srl-collect.js --lab <lab> --node <node> --adj ethernet-1/1.0:<peer> --proto ospf|bgp
--adj maps each local subinterface to the containerlab peer node it faces (so <peer>_iface / <peer>_has_full resolve on any IP plan). The samples above are real captures from an srlinux v24 containerlab lab. One line wires the panel into any GUI: XrayNodePanel.render(el, state).
