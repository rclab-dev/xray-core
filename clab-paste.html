containerlab paste → render (xray-core gallery)

Paste a containerlab topology (.clab.yml) and press Render. Your lab's nodes & links are mapped to an X-Ray diagram — then watch the OSPF state, cut a link, and open a router to see inside.
🔒 Runs entirely in your browser — your topology never leaves the page.
Scope: small FRR labs — 2–3 nodes (2-node link, 3-node path, or 3-node triangle). Bigger graphs aren't mapped yet (X-Ray draws those three shapes). Default protocol = OSPF (? override via the bridge).

inverted-V (3-node path) input is JSON target 
// clab-to-xray.js parses your topology → X-Ray config; clab-xray-bridge.js adds OSPF live state:
var r     = clabToXray.convert(clabYamlText);           // { ok, config } (or { ok:false, reason })
var scene = clabXray.toScene(r.config);                 // real protocol + normal/fault state
var view  = xrayCore.renderTopology('#topo', scene.config, { topology: scene.topos.normal });
view.applyState(scene.states.normal);
view.openDeepDive();                                    // inside the router
Mapping: 2 nodes/1 link → linear · 3 nodes/2 links → path (inverted-V optional) · 3 nodes/3 links → triangle. IP/subnets are synthesized (10.x.0.0/24); real clab IPs would come from containerlab inspect enrichment. Descriptive renderer, not a simulator.
