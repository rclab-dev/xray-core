xray-core gallery

An OSS render engine for live router & network state — driven by the tidy xrayCore API.
xray-core turns a router's state (OSPF/BGP adjacency, routes, interfaces) into a live visualization — an overview topology and an "inside the router" DeepDive cylinder. It is vendor-neutral — FRR is the native, first-class input (feed it FRR directly; Cisco IOS or anything else via a small adapter) — and descriptive (it draws the state you report; it is not a simulator). You drive it with a few calls: renderTopology · applyState · startPolling · openDeepDive.

Bring your own data

Every field xray-core reads is a standard protocol concept, so any router OS works through a small adapter that maps its show output into the documented shapes. See DATA-CONTRACT.md (config / state fields, FRR ↔ Cisco IOS mapping, the two optional seams).
// load: xray-core.js (engine) + xray-api.js (facade), then: var view = xrayCore.renderTopology('#topo', config, { topology, trace }); view.applyState(state); // one snapshot view.startPolling(() => fetch('/api/state').then(r => r.json()), 3000); // or live view.openDeepDive(); // inside the router
Descriptive renderer, not a simulator — it draws the state your feed reports. The live ping/packet animation is hidden in these demos (it is traffic-specific); the engine fully supports it (see DATA-CONTRACT §Seam B). xray-core is the same engine RouteCrushLab runs in production (shared core, no fork).
